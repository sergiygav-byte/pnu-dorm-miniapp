# Оновлення до v8 (контакти, опитування, коментарі санітарки)

## 1. Supabase (обовʼязково)

1. Відкрийте **Supabase → SQL Editor**.
2. Виконайте файл **`supabase/migration_v8.sql`** (один раз).
3. Переконайтесь у Table Editor:
   - `leaders` → колонка **photo**
   - `polls` → **anonymous**, **ends_at**, **end_notified**
   - є таблиця **sanitary_comments**

Якщо ще не запускали **migration_v7.sql** — спочатку його, потім v8 (див. `supabase/ПОРЯДОК-SQL.md`).

## 2. Сайт (GitHub Pages / хостинг)

Завантажте оновлені файли:

- `index.html`
- `database.js`
- `config.public.js` (без змін, якщо ключі вже налаштовані)

```bash
git add index.html database.js supabase/migration_v8.sql api/cron-poll-close.js vercel.json
git commit -m "v8: фото контактів, опитування, коментарі санітарки"
git push
```

## 3. Vercel (бот + cron опитувань)

Якщо API на Vercel:

1. **Deploy** після push (або Redeploy у Dashboard).
2. У **Settings → Cron Jobs** зʼявиться `/api/cron-poll-close` (кожні 15 хв) — закриває прострочені опитування і надсилає push з результатами.
3. Змінні середовища ті самі: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `BOT_TOKEN`, `CRON_SECRET`, `NOTIFY_SECRET`.

## 4. Що змінилось для користувачів

| Розділ | Зміна |
|--------|--------|
| **Контакти** | Фото зберігається і показується в круглому аватарі |
| **Опитування** | Анонімне — відсотки для всіх; публічне — видно хто як проголосував; дата завершення + push після кінця |
| **Санітарка → коментарі** | Коментар додає лише адмін; усі можуть відповідати; відповіді бачить лише адмін |

## 5. Перевірка після оновлення

1. **Контакт:** адмін → редагувати → завантажити фото → зберегти → аватар видно всім.
2. **Опитування:** створити з датою завершення → проголосувати → після дати (або cron) статус «Завершено» + push у бот.
3. **Санітарка:** адмін пише коментар → зʼявляється у списку; студент натискає «Відповісти» → відповідь є у адміна, інші її не бачать.

# Оновлення v3 — що зробити вам (покроково)

У коді вже додано: **чат під скаргою**, **журнал користувачів бота**, **розширені push**, **№ кімнати в контактах**.

Щоб усе запрацювало в хмарі, виконайте **3 кроки** (≈15 хв).

---

## Крок 1. Supabase — SQL міграція

1. Відкрийте [supabase.com](https://supabase.com) → ваш проєкт → **SQL Editor**.
2. Відкрийте файл `supabase/migration_v3.sql` у проєкті на комп’ютері.
3. Скопіюйте **весь** текст → вставте в SQL Editor → **Run**.

Якщо помилка «function already exists» — оновіть файл з репозиторію і запустіть знову (у файлі є `DROP FUNCTION`).

**Перевірка:** у **Table Editor** з’явилась таблиця `complaint_comments`, у `leaders` — колонка `room`, у `bot_subscribers` — `last_seen_at`, `last_webapp_at`.

---

## Крок 2. Оновити сайт (GitHub Pages / Render)

1. Залийте оновлений код на GitHub (`git add .` → `git commit` → `git push`).
2. Дочекайтесь оновлення Pages/Render (1–3 хв).
3. Переконайтесь, що в `config.js` є (якщо використовуєте push):

```js
window.NOTIFY_API_URL = 'https://ВАШ-ПРОЄКТ.vercel.app/api/notify';
window.NOTIFY_SECRET = 'ваш_секрет';
```

Без цих рядків додаток працює, але **Telegram-сповіщення не підуть**.

---

## Крок 3. Передеплой Vercel (бот + push)

1. [vercel.com](https://vercel.com) → проєкт `pnu-dorm-miniapp` → **Deployments** → **Redeploy** (або push у GitHub сам оновить).
2. Змінні середовища мають бути як раніше:
   - `BOT_TOKEN`
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `NOTIFY_SECRET`
   - `WEBAPP_URL` (URL вашого `index.html`)

Після redeploy перевірте в браузері: `https://ВАШ.vercel.app/api/notify` — не обов’язково, webhook має відповідати `ok: true`.

## Що тепер вміє додаток

| Функція | Де дивитись |
|--------|-------------|
| Коментарі під скаргою | Вкладка **Скарги** → під заявкою «Переписка» (адмін пише; автор бачить свої) |
| Хто заходив у бот | Вкладка **Інфо** → блок **Користувачі бота** (лише після входу адміна) |
| Push про внески, звіти, чергування, цілі, опитування, скарги | Усім, хто натиснув **/start** у @pnu_dorm_bot |
| Відповідь автору скарги | Окреме повідомлення в Telegram автору заявки |
| № кімнати в контактах | **Контакти** → додати/редагувати → поле **№ кімнати** |

---

## Важливо для мешканців

Кожен мешканець, який хоче **push у Telegram**, має один раз:

1. Відкрити [@pnu_dorm_bot](https://t.me/pnu_dorm_bot)
2. Натиснути **Start** (/start)

Без цього він не потрапить у список підписників і сповіщення не отримає.

---

## Якщо щось не працює

| Симптом | Рішення |
|--------|---------|
| Немає коментарів / помилка бази | Не виконано `migration_v3.sql` |
| Адмін не бачить «Користувачі бота» | Увійти як адмін; натиснути **Оновити** |
| Немає push | Перевірити `NOTIFY_*` у `config.js` і Vercel; мешканці зробили /start |
| Автор не отримав відповідь на скаргу | Автор мав подати скаргу **з Telegram Mini App** (щоб був `telegram_user_id`) |

Напишіть у чат «застряг на …» + текст помилки — допоможемо.
